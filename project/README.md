# нові функції
- На випадок якщо користувач забув пароль його можна змінити за допомохою додаткового питання
- Реалізував функції редагування транзакції, видалення транзакцію, підрахунок доходів

# основні зміни 
- Додав можливість створювати користувачів
- Регістрацію помістив в нескінчений цикл для зручності
- змінив принцип генерування transaction_id для того щоб транзакції можна було редагувати та видаляти
- виправив exchange_rates бо він був неправильно написаний

### невеликі поправки
- Для деяких input-ів я додав два методи strip та lower щоб користувачу було зручніще вводити дані та щоб уникунути помилок
- Із файлу validation я виніс помилки в відповідні папки
- Виправив функцію та auteficator_user register_user додавши username в значення функції is_user_in_db
- Додав перевірку винятків для функції calculate_budget_remaining оскільки в останій день місяця буде помилка ZeroDivisionError

# нереалізовані функції
- Нажаль не дуже розумію для чого потрібна функцію get_user_input тому не реалізував її
- Також я важаю що функція dayt_until_end_of_week не має сильного застосування в цьому проекті тому також не реалізував її

# принцип роботи 
1. Ввійти або створити користувача
- При реістрації обовязково додати відповідь на питання для зміни паролю
- Exit - вийти з програми
2. Вибрати дію (add, edit, del, calc, budget, balance, report, back)
- Add - додати заробіток або витрату 
- Edit - змінює транзакцію
- Delete - видаляє транзакцію
- Calc - рахує весь дохід або по заданій категорії
- Budget - рахує бюджет на день до кінця місяця
- Balance - показати баланс 
- Report - створити звіт та загрузити його у файл
- Back - повернутися до реістрації
